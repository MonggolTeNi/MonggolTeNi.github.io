Linux常见的命令

1、ls命令
ls -a 除了显示一般文件名外，还会显示隐藏文件
ls -l 需要查看更详细的文件资料
Ls -F 在列出的文件、目录名称后加一符号，对可执行文件添加一个*号，为目录添加一个/号，为符号链接添加一个@号

2、cd 命令
cd - 回到上一次工作的地方
cd ~ 回到家目录
cd/ 回到根目录
cd ../../../../../../../../ 路径较深时，回到根目录
绝对路径和相对路径
相对路径：参照当前目录进行查找
cd ../init.d/
绝对路径：从根目录开始指定，一级一级递归查找，在任何目录下都能进入指定位置
cd /etc/init.d/
目录补全快捷键：Tab

3、mkdir
mkdir建立新目录

4、rmdir
rmdir删除空目录

5、rm命令
rm -f 文件名 强行删除，不给出提示，慎用
rm -i 文件名 删除前询问，无法删除目录
rm -r 目录名 可删除全部目录和子目录和文件，删除前询问

6、cp命令
cp -r 源文件 目的文件 连同源文件中的子目录一同拷贝
cp -b 源文件 目的文件 如果目的文件存在，则自动备份

文件的查找和匹配
28、find命令
find [-path ~] -options [-print -exec -ok]
-path：要查找的目录路径。
~ 表示HOME目录 . 表示当前目录 / 表示根目录

-options常用的：
-name：按照名字查找
-perm：安装权限查找
-prune：不再当前指定的目录下查找
-user：文件属主来查找
-group：文件所属组来查找
-nogroup：查找无有效所属组的文件
-nouser：查找无有效属主的文件
-type：按照文件类型查找
b:块设备文档
d:目录
c:字符设备文档
P：管道文档
l:符号链接文档
f:普通文档

-print 结果标准输出

-exec：对匹配的文件执行该参数所给出的 shell命令。命令形式为command {} \;，注意{}与\;之间有空格
-ok让用户确认是否执行
例：
按名字查找
find . -name ‘[A-Z].txt’ -print 在当前目录和子目录中查找大写字母开头的txt文件
find /etc -name ‘host‘ -print 在/etc及其子目录中，查找host开头的文件
find ~ -name ‘‘ -print 在$HOME目录及其子目录中，查找所有文件
find . -name “out“ -prune -o -name “.txt” -print 在当前目录及子目录中，查找不是out开头的txt文件
按目录查找　　
find . ( -path “./aa” -o -path “./bb” ) -prune -o -name “.txt” -print 　 在当前目录及除aa和bb之外的子目录中查找txt文件
find . ! -name “.” -type d -prune -o -type f -name “*.txt” -print 在当前目录，不在子目录中，查找txt文件

按权限查找
find . -perm 755 -print
按类型查找
find . -type l -print 在当前目录及子目录下，查找符号链接文件 　
按属主与属组查找
find / -user www -type f -print 　属主是www
find / -nouser -type f -print 查找属主被删除的文件
find / -group mysql -type f -print 查找属组mysql的文件
按时间查找
find . -mtime -2 -type f -print 查找2天内被更改过的文件
find . -mtime +2 -type f -print 查找2天前被更改过的文件
find . -atime -1 -type f -print 查找一天内被访问的文件
find . -ctime -1 -type f -print 查找一天内状态被改变的文件
find . -cmin +10 -type f -print 查找10分钟以前状态被改变的文件
按文件新旧查找
find . -newer “aa.txt” -type f -print 查找比aa.txt新的文件
find . ! -newer “aa.txt” -type f -print 查找比aa.txt旧的文件
find . -newer ‘aa.txt’ ! -newer ‘bb.txt’ -type f -print 查找比aa.txt新，比bb.txt旧的文件
按大小查找
find / -size +1M -type f -print 查找超过1M的文件
find . -size 6c -print 查找等于6字节的文件
find . -size -32k -print 查找小于32k的文件
执行文件
find . -name ‘del.txt’ -ok rm {} \; 查找del.txt并删除，删除前提示确认
find . -name ‘aa.txt’ -exec cp {} {}.bak \; 查找aa.txt 并备份为aa.txt.bak

30、sed命令
sed 参数[-nefr] 动作[function] 文件
选项与参数：
-n ：使用安静(silent)模式。在一般 sed 的用法中，所有来自 STDIN 的数据一般都会被列出到
终端上。但如果加上 -n 参数后，则只有经过sed 特殊处理的那一行(或者动作)才会被列出来。
-e ：直接在指令列模式上进行 sed 的动作编辑；
-f ：直接将 sed 的动作写在一个文件内， -f filename 则可以运行 filename 内的 sed 动作；
-r ：sed 的动作支持的是延伸型正规表示法的语法。(默认是基础正规表示法语法)
-i ：直接修改读取的文件内容，而不是输出到终端。

function：
a ：新增， a 的后面可以接字串，而这些字串会在新的一行出现(目前的下一行)～
c ：取代， c 的后面可以接字串，这些字串可以取代 n1,n2 之间的行！
d ：删除，因为是删除啊，所以 d 后面通常不接任何咚咚；
i ：插入， i 的后面可以接字串，而这些字串会在新的一行出现(目前的上一行)；
p ：列印，亦即将某个选择的数据印出。通常 p 会与参数 sed -n 一起运行～
s ：取代，可以直接进行取代的工作！通常这个 s 的动作可以搭配正规表示法！例如 1,20s/old/new/g 就是

注意一下， sed 后面接的动作，必须以 ‘’ 两个单引号括住。
nl /etc/passwd | sed ‘2,5d’ 列印行号，同时将第 2~5 行删除
nl /etc/passwd | sed ‘2d’ 只要删除第 2 行
nl /etc/passwd | sed ‘3,$d’ 要删除第 3 到最后一行
nl /etc/passwd | sed ‘2a drink tea’ 在第二行后(亦即是加在第三行)加上drink tea字样
nl passwd |sed ‘2i drink tea’|head -n 5|nl 在第二行上面添加
nl /etc/passwd | sed ‘2a Drink tea or \ 后按回车

drink beer ‘ 增加两行以上，在第二行后面加入两行字Drink tea or 和drink beer
每一行之间都必须要以反斜杠\ 来进行新行的添加。
nl /etc/passwd | sed ‘2,5c No 2-5 number’ |head（tail） -n 10将第2-5行的内容替换成为『No 2-5 number』，并显示头10行或后10行
nl /etc/passwd | sed -n ‘5,7p’ 仅列出 /etc/passwd 文件内的第 5-7 行
nl /etc/passwd | sed ‘/root/p’ 搜索 /etc/passwd有root关键字的行
nl /etc/passwd | sed -n ‘/root/p’ 使用-n的时候将只打印包含模板的行。
nl /etc/passwd | sed ‘/root/d’ 删除/etc/passwd所有包含root的行，其他行输出
nl /etc/passwd | sed -n ‘/root/{s/bash/blueshell/;p}’ 搜索/etc/passwd,找到root对应的行，执行后面花括号中的一组命令，每个命令之间用分号分隔， 这里把bash替换为blueshell，再输出这行
sed ‘s/要被取代的字串/新的字串/g’
/sbin/ifconfig eth0 利用 /sbin/ifconfig 查询 IP
/sbin/ifconfig eth0 | grep ‘inet addr’ | sed ‘s/^.addr://g’ 将 IP 前面的部分予以删除
/sbin/ifconfig eth0 | grep ‘inet addr’ | sed ‘s/^.addr://g’ | sed ‘s/Bcast.*$//g’ 将 IP 后面的部分予以删除
sed -i ‘s/.$/!/g’ regular_express.txt 将 regular_express.txt 内每一行结尾若为 . 则换成 !
sed -i ‘$a # This is a test’ regular_express.txt 接在 regular_express.txt 最后一行加入『# This is a test』

31、awk命令
awk工作流程：读入有’\n’换行符分割的一条记录，然后将记录按指定的域分隔符划分域，
填充域，$0则表示所有域,$1表示第一个域,$n表示第n个域。默认域分隔符是”空白键” 或 “[tab]键”
cat /etc/passwd |awk -F ‘:’ ‘{print $1}’ 显示以：为分隔符的第1个域
cat /etc/passwd |awk -F ‘:’ ‘{print $1”\t”$7}’
显示/etc/passwd的以：分割的第一个域和第七个域，并在他们之间添加一个tab键的距离
cat /etc/passwd |awk -F ‘:’ ‘BEGIN {print “name,shell”} {print $1”,”$7} END {print “blue,/bin/nosh”}’
显示/etc/passwd的以：分割的第一个域和第七个域，并在他们之间添加一个“，” 在第一行加上name,shell 最后一行添加”blue,/bin/nosh”
awk ‘/root/‘ /etc/passwd 显示/etc/passwd有root关键字的所有行
awk ‘/^root/‘ /etc/passwd 显示/etc/passwd开头为root的所有行
awk -F: ‘/root/{print $7}’ /etc/passwd 显示/etc/passwd有root关键字的行的第七列
awk -F ‘:’ ‘/root/{print “filename:” FILENAME “,linenumber:” NR “,columns:” NF “,linecontent:”$0}’ /etc/passwd
统计/etc/passwd:文件名，每行的行号，每行的列数，对应的root行的完整行内容
显示的内容为：
filename:/etc/passwd,linenumber:1,columns:7,linecontent:root:x:0:0:root:/root:/bin/bash
filename:/etc/passwd,linenumber:11,columns:7,linecontent:operator:x:11:0:operator:/root:/sbin/nologin

36、/etc/inittab文件的作用
Linux启动之后运行的第一个进程init会从/etc/inittab 文件中读取配置并做一些初始化的工作
格式：
id:runlevels:action:process

Id：用 于表示该配置项的id，长度为1~4个字符，id必须在本文件内唯一，如果有重复id，则init启动的时候会报duplicate id的错误，且后面的重复项配置将无效。这边可以将每一条配置项看作数据库中的一条记录，分号分割的部分为各个字段，将id号则可以看作记录的 Primary Key。
Runlevels：程序执行的运行级别。
Action
常见的action

  • initdefault: 该配置项指定系统的默认运行级。
  • respawn: 该配置项所指定的进程如果被结束，则重新启动该进程。
  • wait: 该配置项指定的进程在运行结束之前不要执行下一条配置项。
  • once: 切换到对应的运行级之后，仅执行指定的进程一次。
  • sysinit: 无论以什么运行级启动，系统启动时都要执行该配置项指定的进程。
  • boot: 仅在系统启动时执行一次。
  • bootwait: 仅在系统启动时执行一次，在执行结束之前不执行下一条配置项
  • powerfail: 当接收到UPS的断电通知时执行该项指定的进程。
  • powerwait: 与powerfail相同，但init会等待进程执行结束。
  • powerokwait: 接收到UPS的供电通知时执行。
  • ctrlaltdel: 当用户同时按下 Ctrl+Alt+Del 时执行该项指定的进程。
    process:指定的程序路径和参数
